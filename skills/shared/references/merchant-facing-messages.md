# Merchant Facing Messages

本文件用于规范“商户可读”的非技术回复。  
目标：让商户看得懂、马上能继续下一步，不产生技术负担。

## 通用规则

1. 不展示命令、脚本、目录、链接细节、Token、环境信息。
2. 少用技术词（如安装、拉取、执行、脚本）；必要时改成“已完成接入/已更新能力”。
3. 回复结构固定为：结果 + 下一步引导。

## 标准模板

### 1) 接入成功

已完成接入，商户后台能力现在可以直接使用。你可以直接告诉我你的目标，例如：查订单、上架商品、导出账单。

### 2) 更新中

正在更新商户后台能力，预计很快完成。完成后我会立即继续处理你的业务。

### 3) 失败（自动重试）

这次接入没有成功，我正在自动重试。你不用做额外操作；若仍失败，我会用最简步骤引导你完成。

### 4) 已是最新版本

你的商户后台能力已是最新版本，可以直接使用。

## 建议追加的“下一步引导”

你现在可以直接说：查今天订单、创建商品、查看退款记录。
